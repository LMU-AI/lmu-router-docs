[Trae](https://www.trae.ai/) 编辑器**原生支持接入第三方 API**（Anthropic 兼容协议），无需安装任何插件、无需写配置文件，是所有编辑器里最简单的第三方 API 接入方式。**同时支持 Claude 官方模型与国产大模型（GLM、Qwen、DeepSeek、Kimi 等）**。

<Callout type="info" title="原生方式 vs Codex 插件方式怎么选？">

- **想用 Claude / 国产模型** → 本页的原生方式（Anthropic 协议、GUI 里点几下）
- **想用 GPT-5 / Codex 系列** → 见 [Codex 官方插件](./vscode-plugin)（OpenAI 协议、手写配置文件）

两种方式可以共存，想同时使用两类模型的话两个都配即可。

</Callout>

## 配置步骤

1. 打开 Trae，进入 **设置 → 模型 → 添加模型**
2. **服务商** 选择 **Anthropic**
3. **模型** 选择 **自定义模型**（下拉最下方），在下方「**模型 ID**」输入框中填写模型名（见下方常用模型表）
4. **自定义请求地址** 填写：
   ```
   https://api.lmuai.com/v1/messages
   ```
5. **API 密钥** 填写你在灵眸后台生成的密钥（`sk-` 开头）
6. 点击「确认」保存，切换到该模型即可使用

## 常见问题

### 该选原生方式还是 Codex 插件？

- **原生方式（本页）**：走 Anthropic 协议、GUI 里点几下就好，适合想用 **Claude / GLM / Qwen** 等模型的用户
- **[Codex 插件方式](./vscode-plugin)**：走 OpenAI 协议、需要手写配置文件，适合想用 **GPT / Codex** 系列的用户

两种方式可以共存 —— 想同时用两类模型的话，两个都配即可。

### 请求地址结尾为什么要带 `/v1/messages`？

Trae 的「自定义请求地址」需要填**完整端点**（不像 Chatbox 等工具会自动拼路径）。Anthropic 协议对话端点固定是 `/v1/messages`，所以完整地址是 `https://api.lmuai.com/v1/messages`。

### 模型 ID 从哪里查？

完整可用列表见 [模型广场](../guide/models)。点任意卡片即可复制模型 ID，直接粘贴到 Trae 的「模型 ID」输入框。

### 添加后没反应 / 报错？

- 服务商是否选了 **Anthropic**（不是 OpenAI）
- 模型下拉是否选了「自定义模型」再手动输入的模型 ID
- 请求地址是否以 `/v1/messages` 结尾
- API Key 是否是灵眸后台生成的 `sk-` 开头密钥

保存后重启一次 Trae 客户端。
